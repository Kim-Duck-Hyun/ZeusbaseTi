Intro - top

This app template is designed for Appcelerator Titanium Mobile projects (iOS only). If you are not familiar with the Titanium platform then refer to the Getting Started with Titanium guide.

The goal of this template is to give you a foundation for your store application. This template includes all the windows required for a user to browse products and manage a shopping cart.

There's even code to handle transactions via PayPal (PayPal Module v1.4 required). Here is a great video on installing the PayPal module. You will also need to configure the PayPal settings in the configuration file. To handle order fulfillment you will have to add hooks to your backend/serverside system (See Order fulfillment).

Setup - top

This template contains the core files required to setup your store app. Setup is super easy! Simply create a new Titanium project. Delete all the files in the project Resources folder. Then copy all the files from the StoreApp-Template/Resources folder over to the project Resources folder.

Now all you need to do is customize your application and add products. You can customize using the Resources/configuration.js file. Open the file up in your editor and update the values for each key.

Here's a breakdown of each configuration setting:

{
	// Background Color for the Spotlight section
	"SPOTLIGHT_BG_COLOR" : "#191919",

	// Background Image for the Spotlight section (not required)
	"SPOTLIGHT_BG_IMAGE" : null,

	// Default thumb image
	"PRODUCTS_DEFAULT_THUMB_IMAGE" : "path/to/default_thumb.png",

	// Default main image
	"PRODUCTS_DEFAULT_MAIN_IMAGE" : "path/to/default_main.png",

	// Default spotlight image
	"PRODUCTS_DEFAULT_SPOTLIGHT_IMAGE" : "path/to/default_spotlight.png",

	// Info screens (New format in v1.1)
	"INFO_SCREENS" : [
		Row One
		{
			"TITLE" : "About Us",
			"URL"   : "http://www.your-domain.com/path/to/about_page"
		},
		Row Two
		{
			"TITLE" : "Contact Us",
			"URL"   : "http://www.your-domain.com/path/to/contact_page"
		}
	],

	// PayPal App ID Issued by PayPal... set to "null" for testing (REQUIRED)
	"PAYPAL_APP_ID" : "null",

	// PayPal payment recipient email (so you get paid)
	"PAYPAY_RECIPIENT" : "add_email_here",

	// PayPal transaction type (HARD_GOODS, SERVICE or PERSONAL)
	"PAYPAL_TRANSACTION_TYPE" : "HARD_GOODS",

	// PayPal payment environment (LIVE, SANDBOX or NONE)
	"PAYPAL_ENVIRONMENT" : "SANDBOX",

	// Whether or not to select/send PayPal shipping information
	"PAYPAL_ENABLE_SHIPPING" : false,

	// Merchant Name used in PayPal transaction (default to app name)
	"MERCHANT_NAME" : Ti.App.name,

	// URL for product list on server
	"PRODUCT_LIST_URL" : "http://www.your-domain.com/path/to/Products.js",

	// Successful Payment Message (New in v1.1)
	"PAYMENT_SUCCESS_MESSAGE" : {

		// Title for success message
		"TITLE"   : "Payment successful!",

		// Success message to display
		"MESSAGE" : "Thank You for your purchase!"
	}
}

Next step is to add products to your store application. To do this you need to create a product list file. The file must be in JSON format. There is an example file in the Resources/data/Products.js.

Here's a breakdown of a product entry:

{
	// Product Name
	"name" : "Magical Unicorn",

	// Product ID
	"id" : "101010",

	// Price
	"price" : 999999.99,

	// Groups are used to orginize products in the store
	"groups" : {

		// Add to spotlight section.
		// Requires spotlight image in "imgs" settings)
		"spotlight" : true,

		// Add to featured section
		"featured" : false,

		// Assign product to a category
		"category" : "Pets"
    },

	// "desc" is used in various places to
	// describe or better yet sell your product
	"desc" : {

		// A short description of the product
		"short" : "An awesome Unicorn that's Magical!",

		// A long description of the product. You can add HTML tags.
		"long" : "<p>The <strong>unicorn</strong> is a legendary magical  animal from European folklore that resembles a white horse with a large, pointed, spiraling horn projecting from its forehead, and sometimes a goat's beard.</p><p>Buy One Now! It's Magical!!</p>"
	},

	// Images to show show off the product ("main" and "thumb" are required).
	"imgs" : {

		// URL for the main image of this product (default size = 133x100)
		"main" : "http://your-domain.com/path/to/main_img.png",

		// URL for the thumbnail image of this product (default size = 75x56)
		"thumb" : "http://your-domain.com/path/to/thumb.png",

		// URL for the spotlight image of this product. Only required if this
		// product is included in the spotlight group (default size = 322x210)
		"spotlight" : "http://your-domain.com/path/to/spotlight_img.png"
    },

	// Options for product (ex. size, color, etc...)
	"options" : [
		{
			// Name of option
			"name" : "color",

			// Enable multiple selections
			"multiple" : false,

			// List of selections for this option
			"selections" : ["White", "Brown", "Black"]
		}
}

That's it!! Your Store App Template is ready to get hooked up to your backend/server fulfillment system. Use the methods in the next section to send order information to your system.

Order fulfillment - top

You must connect the application with your order fulfillment system before you ship your new store app. This might be a simple email script or a complex third party platform. Either way you'll need the order information

The Resources/ProcessOrder.js file contains the handler functions you can use to quickly integrate your backend with the app. More details on each function below.

Name Arguments Description
ProcessOrder.success()
order (JSON)
transactionId (String)
 This function is required to fulfill orders. The order JSON object contains all the items the user has purchased. The transactionId is what PayPal has returned for a successful order.
ProcessOrder.cancelled()
order (JSON)
 This is called when the order was cancelled for any reason. The order JSON object contains all the items the user had in their cart before they cancelled the order. You can use this for analytics.
ProcessOrder.failed()
order (JSON)
errorCode (String)
errorMessage (String)
 This is called when the order fails for any reason. The order JSON object contains all the items the user tried to purchased. The errorCode and errorMessage can be used to track down why the order failed and alert the user.
